Accessible e‑fleet for Linz – from idea to running prototype

We built a wheelchair‑first e‑mobility service that connects to Linz AG public transport. Book an accessible e‑vehicle, see real‑time departures and simulate simple dispatching – all in one lightweight demo.

Tech: Node.js · Express · Vanilla JS · Linz AG EFA API · Accessibility‑first UI · n8n/WhatsApp/Voice (concept)

Hackathon challenge

Design an AI‑powered mobility solution that genuinely improves accessibility for people with disabilities in Linz.

Our focus

Wheelchair users who struggle with standard taxis, ride‑sharing and apps that ignore ramps, secure spaces and assistance needs.

What we deliver

A working booking flow, real‑time departure integration, basic e‑fleet dispatch logic and a plan for voice/WhatsApp access.

Problem: door‑to‑door accessibility is broken

Mobility gaps for wheelchair users

  • Standard taxis and ride‑sharing often lack ramps or safe wheelchair spaces.
  • Public transport apps rarely capture number of wheelchair users or assistance level.
  • Planning door‑to‑door trips with transfers is time‑consuming and error‑prone.

Information and interface barriers

  • Many UIs are not accessible – low contrast, small fonts, poor screen‑reader support.
  • People who cannot comfortably use a keyboard or screen are often excluded.
  • Voice‑based access (phone, WhatsApp) is rarely integrated with live mobility data.

Solution: Linz Accessible E‑Fleet

A prototype booking portal for a wheelchair‑accessible e‑fleet that integrates tightly with Linz AG’s Elektronische Fahrplanauskunft (EFA) API.

Accessible booking

People can book rides for today or tomorrow, explicitly stating wheelchair count, assistance level and notes (service dog, medical equipment, transfers, …).

Public transport integration

Pickup and destination use the Linz AG Stop‑Finder. After booking, the app calls the Departure Monitor to show live departures at the pickup stop – ideal for coordinating transfers.

Accessibility‑first UI

High‑contrast mode, font‑size controls, semantic HTML, ARIA roles, keyboard navigation and a DE/EN toggle in the main app view.

Fake dispatch logic

Bookings are automatically assigned to one of three demo e‑vehicles using simple load balancing that respects wheelchair capacity.

How it works (prototype flow)

  1. User books online. A web form captures personal data, origin/destination (via Linz AG Stop‑Finder), date/time and accessibility needs.
  2. Backend validates and assigns. Node/Express enforces “today or tomorrow only” and assigns a demo e‑vehicle based on upcoming load and wheelchair capacity.
  3. Live departures for transfers. The app calls Linz AG’s XML_DM_REQUEST to display the next departures from the pickup stop with mobility‑impaired options activated.
  4. Operator view. A simple dashboard shows all upcoming bookings in the next 24 hours, including route, wheelchairs, assistance level and assigned vehicle.

Architecture & tech stack

Stack

  • Node.js + Express backend
  • Static HTML/CSS/Vanilla JS frontend
  • Linz AG EFA XML Schnittstelle (JSON output)
  • In‑memory storage for hackathon simplicity

Key APIs

  • /api/stops → proxies to XML_STOPFINDER_REQUEST
  • /api/departures → proxies to XML_DM_REQUEST with mobility options
  • /api/bookings → booking creation & fake dispatch
  • /api/bookings (GET) → upcoming trips for operator view

Next: WhatsApp & voice for full accessibility

Not everyone can or wants to use a browser. The same APIs can power a voice‑first booking experience via WhatsApp, n8n, OpenAI and ElevenLabs.

WhatsApp voice entry

A user sends a voice note to a WhatsApp number (Twilio). n8n receives the audio and uses OpenAI Whisper to transcribe it.

AI‑assisted booking

OpenAI Chat extracts origin, destination, date/time and accessibility needs, then calls our /api/bookings endpoint to create a real booking.

Spoken confirmation

ElevenLabs generates a short confirmation audio message which is sent back to the user on WhatsApp together with a text summary.

How to run the demo locally

  1. Install dependencies
    npm install
  2. Start the server
    npm run dev or npm start
  3. Open the app
    Go to http://localhost:3000/index.html to use the booking prototype.

The intro page you are viewing can be opened as http://localhost:3000/intro.html. From there, click “Open live prototype”.
